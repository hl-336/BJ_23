web端自动化测试

需求变动不频繁
项目界面变动过于频繁会导致脚本没有办法重复运行
项目周期要长
要实现UI自动化其花费的时间常规来讲是手工测试3倍(至少半年以上项目)
项目需要回归测试
模拟手工测试的测试,而手工测试是项目周期中必不可少的阶段,如果没有版本迭代带来的回归
测试,那引入UI自动化无意义

import time
from selenium import webdriver
# 打开浏览器后先窗口最大化
driver = webdriver.Chrome()
driver.get("http://127.0.0.1/Home/User/login.html")
# 实现窗口最大化
driver.set_window_size(2000, 5000)
# 使用xpath定位tpshop首页登陆超链接，并执行点击
driver.find_element_by_partail_link_text("超链接的局部或者完整文本内容")
# 使用id定位登陆页面用户名输入框，输入用户名
driver.find_element_by_id("username").send_keys("13811111111")
# 使用name定位登陆页面密码输入框，输入密码
driver.find_element_by_name("password").send_keys("1234576")
# 使用class定位定位登陆页面验证码输入框，输入验证码
driver.find_element_by_class_name("text_cmu").send_keys("1234576")
# 使用xpath定位登陆按钮执行点击
driver.find_element_by_xpath("//*[contains(@placeholder,'登')]").click()
# 打印登陆后页面标题和地址信息
print(driver.title)
print(driver.current_url)
